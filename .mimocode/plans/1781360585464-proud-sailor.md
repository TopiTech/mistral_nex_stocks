## P0 Fixes

### Fix #1: Shutdown Token Race Condition
**Files**: `app.py:490-583`, `routes/api_system.py:464-543`
**Problem**: Token rotation happens AFTER shutdown thread starts, creating a race window.
**Fix**:
- Rotate token BEFORE spawning shutdown thread in `api_shutdown()`
- Add threading lock around `SHUTDOWN_TOKEN` and `SHUTDOWN_TOKEN_USED` state access
- Move `_rotate_shutdown_token()` call to before `shutdown_thread.start()`

### Fix #2: CSRF Exemption for `api_add_stock_ext`
**Files**: `app.py:1023`, `routes/api_stocks.py:646-696`
**Problem**: Only relies on `X-MNS-Extension-Request` header which can be forged.
**Fix**:
- Add `_is_allowed_shutdown_origin(request)` check (already present at line 657, but only as fallback)
- Strengthen: require BOTH the custom header AND origin validation
- Alternatively, require the shutdown token for extension requests too

## P1 Fixes

### Fix #3: Rate Limiter Localhost Bypass
**File**: `route_helpers.py:99-157`
**Problem**: Localhost requests completely bypass rate limiting.
**Fix**:
- Remove the localhost bypass for rate limiting
- Keep a higher limit for localhost (e.g., 2x the normal limit) instead of unlimited
- OR: Apply rate limiting to all requests including localhost

### Fix #4: SSE Stream Authentication
**File**: `routes/api_stocks.py:803-846`
**Problem**: No `_is_local_request` check on SSE endpoint.
**Fix**:
- Add `@rate_limit(max_requests=10, window_seconds=60)` decorator
- The endpoint already uses `stream_with_context` which is good
- Consider adding `_is_local_request` check (but this may break legitimate use)

### Fix #5: Chrome Extension CSP Hardening
**File**: `chrome_extension/manifest.json:17-19`
**Problem**: Missing `style-src`, overly broad `connect-src`.
**Fix**:
- Add `style-src 'self'`
- Restrict `connect-src` to specific port: `http://127.0.0.1:5000 http://localhost:5000`
  - Note: The port may vary, so this needs to match `DEFAULT_BACKEND_PORT`
  - Alternative: keep `*` but document the tradeoff
