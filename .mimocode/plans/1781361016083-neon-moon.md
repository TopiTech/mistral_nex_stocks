# Security Code Review Plan: Mistral NeX Stocks

## Executive Summary

Multi-component stock analysis app (Flask + Chrome Extension + Native Host) with **strong baseline security** (4.5/5). No critical vulnerabilities. 2 HIGH, 5 MEDIUM, 7 LOW findings across git hygiene, CSP, and defense-in-depth gaps. Financial disclaimer present in UI but missing from API payloads. Dependency posture is healthy — Flask >=3.1.3 avoids known CVEs.

**Scope**: Flask backend, Chrome/Edge MV3 extension, Windows Native Messaging Host, CI pipeline
**Method**: Manual code review + tooling (bandit, pip-audit, flake8, mypy, pytest)
**Total findings**: 2 HIGH, 5 MEDIUM, 7 LOW, 13 INFORMATIONAL (positive)

## Findings by Severity

### CRITICAL: None identified

### HIGH (2)

#### H1: Sensitive Files Tracked in Git
| | |
|---|---|
| **Component** | Git repository |
| **Files** | `user_stocks.json`, `user_stocks.json.bak` (tracked); `.roo/mcp.json` (tracked despite `.gitignore`) |
| **Description** | User portfolio data and MCP config are tracked in git. The `.gitignore` excludes `config.json` and `.mns_shutdown_token` but has no rule for `user_stocks.json*`. `.roo/` is gitignored (line 11) but `.roo/mcp.json` was added before the rule. |
| **Risk** | Portfolio data exposure in git history; credential/config leakage if repo is shared. |
| **Recommendation** | 1. Add `user_stocks.json*` to `.gitignore`. 2. `git rm --cached .roo/mcp.json user_stocks.json user_stocks.json.bak`. 3. Consider `git filter-branch` or BFG to purge from history. 4. Add pre-commit hook to block tracked sensitive files. |

#### H2: Native Host Extension ID Bypass When Allowed Origins Empty
| | |
|---|---|
| **Component** | Native Host |
| **File** | `native_host/native_host.py:162-163` |
| **Description** | `_validate_extension_id()` calls `_load_allowed_manifest_origins()` and skips validation when `allowed_ids` is empty (line 163: `if allowed_ids and extension_id not in allowed_ids`). If the manifest file is missing or unparseable, any extension with valid ID format is accepted. |
| **Risk** | Any Chrome extension could connect to the native host if the manifest is corrupted or deleted. |
| **Recommendation** | Fail closed: reject connection when `allowed_ids` is empty rather than bypassing validation. Add a hardcoded fallback ID or fail with explicit error. |

### MEDIUM (5)

#### M1: Extension CSP Blocks Google Fonts
| | |
|---|---|
| **Component** | Chrome Extension |
| **File** | `chrome_extension/manifest.json:17-18`, `chrome_extension/popup.html:7` |
| **Description** | `popup.html` loads `fonts.googleapis.com` via `<link>` but CSP is `style-src 'self'` — no external sources allowed. Fonts silently fail to load. |
| **Risk** | Degraded UX (font fallback). Minor — no security impact. |
| **Recommendation** | Either: (a) add `https://fonts.googleapis.com` to `style-src` and `https://fonts.gstatic.com` to `font-src`, or (b) remove the external `<link>` and rely on system fonts. Option (b) is more secure. |

#### M2: Rate Limiter Resets on Restart
| | |
|---|---|
| **Component** | Flask Backend |
| **File** | `route_helpers.py:63-67` |
| **Description** | Rate limit store is in-memory (`_rate_limit_store` dict). Restarting the server clears all counters. No persistence. |
| **Risk** | An attacker can bypass rate limits by triggering a restart (e.g., via `/api/shutdown` with valid token, though token is single-use). |
| **Recommendation** | Acceptable for personal-use app. If hardened: persist to SQLite or Redis. At minimum, document the limitation. |

#### M3: `.mns_shutdown_token` Accidental Commit Risk
| | |
|---|---|
| **Component** | Project Root |
| **File** | `.gitignore:22-23` |
| **Description** | `.mns_shutdown_token` and `.mns_shutdown_token.used` are gitignored. However, the token is created at runtime in the project root — if someone does `git add -A` quickly, it could be staged before gitignore takes effect. |
| **Risk** | Low — gitignore covers it, but `git add -A` timing race exists. |
| **Recommendation** | Add a `.gitignore` comment noting this risk. Consider storing the token in a `tmp/` subdirectory instead. |

#### M4: bandit Report Artifact Tracked in CI
| | |
|---|---|
| **Component** | CI Pipeline |
| **File** | `.github/workflows/ci.yml:70` |
| **Description** | `bandit_report.txt` is generated during CI. The `*.txt` gitignore (line 7) is overridden by `!requirements.txt` / `!requirements-locked.txt`, but the CI artifact is uploaded via `actions/upload-artifact` (not committed). However, the `.gitignore` `*.txt` rule may interfere with local bandit runs. |
| **Risk** | Low — artifact is in GitHub Actions, not git. But confusing for local dev. |
| **Recommendation** | Ensure `.gitignore` `*.txt` rule doesn't block local `bandit_report.txt` generation. Add explicit ignore if needed. |

#### M5: DaemonThreadPoolExecutor Global Thread Monkey-Patching
| | |
|---|---|
| **Component** | Flask Backend |
| **File** | `app_state.py:475-499` |
| **Description** | `DaemonThreadPoolExecutor._adjust_thread_count()` temporarily replaces `threading.Thread` globally (line 486: `threading.Thread = daemon_thread`). While wrapped in try/finally, a failure in `super()._adjust_thread_count()` could leave the global in a broken state if another thread calls `Thread()` during the window. |
| **Risk** | Theoretically could affect other threads during the monkey-patch window. In practice, the window is very small and failures are unlikely. |
| **Recommendation** | Document the tradeoff. Consider subclassing `threading.Thread` directly instead of monkey-patching. |

### LOW (6)

| **Risk** | Low — pins prevent accidental upgrades but also block patches. |

#### L4: No [tool.ruff] or [tool.black] Config
| | |
|---|---|
| **Component** | Project Config |
| **File** | `pyproject.toml` |
| **Description** | Only bandit config exists in `pyproject.toml`. No formatter/linter config. CI uses flake8 + pylint but no auto-formatter. |
| **Risk** | None (code quality, not security). |
| **Recommendation** | Add `[tool.ruff]` for consistent formatting. Low priority. |

#### L5: B101 (assert) Skipped Globally in Bandit
| | |
|---|---|
| **Component** | Security Tooling |
| **File** | `pyproject.toml:4` |
| **Description** | Bandit skips B101 (assert) globally. `assert` in production code can be stripped with `-O` flag, potentially masking bugs. |
| **Risk** | Low — app is not run with `-O`. But assert-based checks would silently disappear. |
| **Recommendation** | Use explicit `if not condition: raise` instead of `assert` in non-test code. Narrow the B101 skip to only `tests/` directory. |

#### L6: CSP Allows All Localhost Ports + CDN in script-src
| | |
|---|---|
| **Component** | Flask Backend |
| **File** | `app.py:280,286` |
| **Description** | `connect-src` includes `http://localhost:* http://127.0.0.1:*` — allows connection to any port on localhost. Additionally, `script-src` includes `https://cdn.jsdelivr.net` — a third-party CDN that, if compromised, could inject malicious scripts (nonce protection mitigates this for Flask-served pages). |
| **Risk** | Localhost wildcard: compromised page could connect to other local services. CDN: supply-chain risk, though nonces limit impact to pages that explicitly include CDN scripts. |
| **Recommendation** | If hardened: restrict connect-src to specific backend port. For CDN: consider self-hosting JS assets. Document the tradeoff. |

#### L7: `_is_local_request` Trusts X-Forwarded-For
| | |
|---|---|
| **Component** | Flask Backend |
| **File** | `app_helpers.py:367-381` |
| **Description** | When behind a reverse proxy, `_is_local_request()` checks the first `X-Forwarded-For` IP. However, this header is trivially spoofable by any client. The function does defense-in-depth by rejecting if ANY forwarded IP is non-local, but the initial trust assumption is weak. |
| **Risk** | If deployed behind a reverse proxy on a shared network, an attacker on the same network could spoof `X-Forwarded-For` to bypass localhost checks. Acceptable for personal localhost-only use. |
| **Recommendation** | Document that this function is designed for localhost-only deployment. If ever exposed behind a proxy, use `ProxyFix` with `x_for=1` and ensure the proxy strips client-supplied headers. |

## Financial Data App Specific Concerns

### F1: Disclaimer Missing from API Response Payloads
| | |
|---|---|
| **File** | `templates/index.html:251-265` (present), API endpoints (absent) |
| **Description** | Financial disclaimer exists in the web UI footer (bilingual JA/EN). However, API responses from `/api/analysis/*` endpoints that return AI-generated analysis contain no disclaimer text. If a consumer relies on the API data without viewing the UI, they miss the warning. |
| **Recommendation** | Add a `"disclaimer"` field to analysis response payloads, or include it in a response header. |

### F2: yfinance TLS Fingerprint Impersonation
| | |
|---|---|
| **File** | `app_state.py:356-359` |
| **Description** | `curl_requests.Session(impersonate="chrome")` makes yfinance requests appear as Chrome browser. This is for bypassing yfinance's bot detection. |
| **Ethical Note** | This is a gray area. yfinance ToS may prohibit automated access. For personal-use stock tracking, this is common practice. Document the intent. |

### F3: SSE Stream Leak Potential
| | |
|---|---|
| **File** | `routes/api_stocks.py:806-850` |
| **Description** | SSE endpoint has rate limiting (10/min) but no `_is_local_request` check. The `GeneratorExit` handler (line 841) properly cleans up via `unlisten(q)`. However, if the server restarts mid-stream, clients may hang until timeout. |
| **Recommendation** | Acceptable for personal use. Consider adding a max connection duration. |
