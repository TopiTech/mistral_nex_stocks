# Mistral NeX Stocks – 総合コードレビュー（詳細版）

## プロジェクト概要
Flask + Chrome拡張(MV3) + Windows Native Host構成のリアルタイム株価情報・AI分析Webサービス。
- **バックエンド**: Python 3.9+ / Flask 3.1 / yfinance / Mistral AI SDK / DDGS / LangSearch
- **フロントエンド**: Vanilla JS (11ファイル) / Chart.js / SSE
- **拡張機能**: Chrome Extension Manifest V3 / Native Messaging
- **データソース**: Yahoo Finance (yfinance), DuckDuckGo Search, LangSearch, Google Trends (pytrends), RSS feeds

## 1. セキュリティレビュー（詳細）

### 1.1 Chrome拡張セキュリティ（Manifest V3準拠）

**確認結果（Chrome公式ドキュメント照合）**:
- MV3の `content_security_policy` は辞書形式（`extension_pages` + `sandbox`）が正しい ✅
- リモートホストコード未使用（全てバンドル内） ✅
- `script-src 'self'` で外部スクリプト実行を禁止 ✅
- `host_permissions` は `http://127.0.0.1/*` と `http://localhost/*` のみ ✅

**課題**:
1. **`connect-src` の `http://localhost:*` がポート制限なし** (`manifest.json:18`)
   - MV3 CSPの `connect-src` が `http://localhost:*` でポートを指定していないため、任意のローカルポートへの接続が許可される。ポートを明示すべき（例: `http://localhost:5000`）。

### 1.2 バックエンドセキュリティ

**CSP/CSRF**:
- CSP nonce がリクエストごとに生成される（`app.py:294-296`） ✅
- `csrf.js` が `fetch` をパッチして `X-CSRFToken` ヘッダーを自動付与（`csrf.js:24-51`） ✅
- `api_add_stock_ext`, `api_shutdown`, `api_csp_report` がCSRF exempt（`app.py:1028-1030`） - 設計上の判断として妥当

**APIキー管理**:
- keyring > DPAPI > 平文 のフォールバック階層 ✅
- `MNS_ALLOW_PLAINTEXT_SECRETS=1` 環境変数で平文許可 ✅
- ログ出力はフィンガープリント（SHA256ハッシュ）のみ ✅
- APIキー検証で最小長チェック（`_is_valid_api_key`） ✅

**入力検証**:
- `is_valid_symbol` でパストラバーサル・SQLインジェクション対策 ✅
- `SYMBOL_PATTERN` で正規表現バリデーション ✅
- `MAX_JSON_SIZE` (1MB) でJSONボディサイズ制限 ✅
- `MAX_CONTENT_LENGTH` (16MB) でFlaskコンテンツ長制限 ✅
- `parse_non_negative_float` で数値入力の厳格な検証 ✅

**セキュリティヘッダー**:
- `X-Content-Type-Options: nosniff` ✅
- `X-Frame-Options: DENY` ✅
- `Referrer-Policy: strict-origin-when-cross-origin` ✅
- `Strict-Transport-Security` (HTTPS時のみ) ✅

**Native Host**:
- アクションのホワイトリスト検証（`ALLOWED_ACTIONS`） ✅
- 拡張IDのフォーマット検証（32文字小文字英数字） ✅
- マニフェストの `allowed_origins` と照合 ✅
- stdout が stderr にリダイレクト（プロトコル保護） ✅
- メッセージサイズ制限（`MAX_MESSAGE_BYTES` = 1MB） ✅

### 1.3 セキュリティ上の改善点

#### 高優先度
1. **Native Hostの `get_shutdown_token` がトークンを平文返却** (`native_host.py:268-300`)
   - シャットダウントークンが `send_message` で平文JSONとして返される。Chrome拡張の `background.js` で `mnsShutdownToken` 変数に格納される。Service Workerのメモリ内であるため実用上問題ないが、`console.warn` でトークン関連のエラーログが出る可能性あり。

2. **`_is_local_request` のX-Forwarded-For検証** (`app_helpers.py:367-384`)
   - プロキシ環境では `X-Forwarded-For` ヘッダー偽装が可能。現在はlocalhost専用だが、将来プロキシ設定を追加する場合は信頼できるプロキシの設定が必要。

#### 中優先度
3. **CSPの `connect-src` ポート制限** (`app.py:286`)
   - `http://localhost:*` → `http://localhost:5000` に限定すべき（現在のポート設定に合わせて）

4. **`config.json.bak` が `.gitignore` に含まれていない** (`.gitignore`)
   - `config.json` はgitignoreされているが、`.bak` ファイルが含まれる可能性がある。`.gitignore` に `config.json.bak` を追加推奨。（確認: `.gitignore:2` に `config.json.bak` が記載されている ✅）

```

app_bg.py           → バックグラウンド処理（yfinance取得、SSE補間）
```

### 2.2 定数の重複問題（最重要改善点）

**同一定数が4ファイル以上に分散している**:

| 定数名 | app.py | app_bg.py | route_helpers.py | services/ai_service.py |
|--------|--------|-----------|------------------|----------------------|
| `MISTRAL_API_TIMEOUT_SEC` | :737 | - | :33 | :20 |
| `MISTRAL_MIN_INTERVAL_SEC` | :738 | - | :34 | :21 |
| `MISTRAL_API_KEY_MIN_LENGTH` | :739 | - | :35 | - |
| `YFINANCE_TIMEOUT_BATCH` | :801 | :34 | :37 | - |
| `YFINANCE_TIMEOUT_SINGLE` | :804 | :35 | :38 | - |
| `MAX_JSON_SIZE` | :585 | - | - | - |
| `PORTFOLIO_SHARES_MAX` | :828 | - | :42 | - |

**影響**: 環境変数で値を変更した場合、ファイルによって異なる値が使われる可能性がある。

### 2.3 `app.py` の残存ロジック

Blueprint移行後も `app.py` に以下のロジックが残存:
- `BackendLogFilter` / `PollingFilter` クラス（`app_state.py` にも重複）: `app.py:349-454`
- `IMPORTANT_INFO_PATTERNS` 定数（`app_state.py` にも重複）: `app.py:336-346`
- `_get_backend_port` 関数（`app_helpers.py` にも重複）: `app.py:317-332`
- `_market_news_freshness_policy` / `_filter_recent_market_news_items`: `app.py:754-792`

### 2.4 `AppState` の動的属性解決

```python
# app_state.py:849-883
def __getattr__(self, name):
    for group_name in ("execution", "market", "ai", "cache"):
        group = getattr(self, group_name, None)
        if group is not None and hasattr(group, name):
            return getattr(group, name)
    raise AttributeError(...)
```

**問題点**:
- IDEの補完が効かない
- 型チェックが不可能
- デバッグ時に属性の所在が不明
- `setattr` 側も同様の問題

### 2.5 依存関係の循環

`app.py` → `routes/*.py` → `app.py` の循環を回避するため、`app.py` のルート定義がBlueprintに移行されたが:
- `app.py:1017-1025` でBlueprintをimportして登録
- `routes/api_system.py:442` で `from app import _consume_shutdown_token, _rotate_shutdown_token` と `app.py` を再import
- `routes/*.py` 全ファイルで `from app_helpers import ...` と `app_helpers.py` を大量import
