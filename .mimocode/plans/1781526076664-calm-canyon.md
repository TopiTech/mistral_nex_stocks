# Code Review Plan — Mistral NeX Stocks v3

> "このプロジェクトのコードレビューを詳細に行って。web検索を用いて、最新ドキュメントを確認しながらレビューを行って。レビューでは機能拡張、改善、UI改善、バックエンド不具合修正など様々な観点からレビューを実施して。"

## Scope

Full-stack code review covering: Security, Architecture, Backend, Frontend, UI, Tests, Dependencies, Code Quality. Each section will reference latest documentation via web search where applicable.

## Review Categories

### 1. Security (High Priority)
**Files:** `app.py`, `config_utils.py`, `app_helpers.py`, `routes/api_system.py`, `native_host/native_host.py`

- [ ] CSP nonce injection fragility (`app.py:658-667` — `re.sub` on policy string)
- [ ] Shutdown token security: single-use rotation, file permissions, timing attack resistance
- [ ] CSRF exemptions review (`app.py:878-880` — api_csp_report, api_shutdown, api_add_stock_ext)
- [ ] Secret storage chain: keyring → DPAPI → plaintext fallback security
- [ ] Flask session config: `SESSION_COOKIE_PARTITIONED`, `SESSION_COOKIE_SECURE`
- [ ] `X-Forwarded-For` trust in `_is_local_request()` — spoofing risk behind reverse proxy
- [ ] Native host extension ID validation (fail-closed when allowed_origins empty)
- [ ] Verify: latest Flask 3.1 security defaults, latest Mistral SDK auth patterns

### 2. Architecture (Medium Priority)
**Files:** `app_state.py`, `constants.py`, `route_helpers.py`, `trend_sources.py`

- [ ] `DaemonThreadPoolExecutor` defined TWICE with different implementations:
  - `app_state.py:455-466` — sets daemon flag on existing threads
  - `trend_sources.py:54-75` — monkey-patches `threading.Thread` globally (THREAD-SAFETY RISK)
- [ ] `AppState.__getattr__`/`__setattr__` dynamic delegation — impacts IDE support
- [ ] `_BASE_ALLOWED_CORS_ORIGINS` duplicated: `constants.py:97-100` and `app.py:603-606`
- [ ] `SYMBOL_PATTERN` duplicated: `app.py:609` and `app_helpers.py:50`
- [ ] Module dependency graph: identify circular import risks
- [ ] Region comments (`#region`/`#endregion`) — consistency across files

### 3. Backend — Bug Fixes & Robustness
**Files:** `app_bg.py`, `app_helpers.py`, `route_helpers.py`, `services/ai_service.py`

- [ ] `interpolate_value()` uses `random.uniform` — not deterministic, affects test reproducibility
- [ ] USDJPY hardcoded fallback to 150.00 when yfinance fails — stale rate risk
- [ ] `fetch_index_data()` 3-level fallback chain: ticker history → yf.download → hardcoded default
- [ ] `bg_interpolate_loop` market status caching: 60s check interval may miss fast market open/close transitions
- [ ] Thread pool exhaustion risk: `sync_refresh_executor` has `max_workers=1` — concurrent sync requests may block
- [ ] `save_user_stocks()` file replace outside lock — race condition between `os.replace` and `last_modified_ns` update
- [ ] Rate limit store memory leak: `_cleanup_rate_limit_store` only runs on decorated endpoint calls

### 4. Backend — Feature Extensions
**Files:** `services/ai_service.py`, `services/search_service.py`, `trend_sources.py`

- [ ] Mistral SDK: check if `reasoning_effort` parameter is available on current model (`mistral-medium-3.5`)

- [ ] LangSearch integration: retry logic (503 only), timeout handling
- [ ] DDGS query length limit (500 chars) — no truncation warning to user
- [ ] Google Trends RSS vs. API — reliability comparison for 2026
- [ ] Wikipedia pageviews API: rate limiting considerations
- [ ] GDELT API: `HybridRel` sort mode — verify stability

### 5. Frontend — UI Improvements
